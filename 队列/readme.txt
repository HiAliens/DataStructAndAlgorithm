顺序队列是一种限制性线性表，采用先进先出规则，队尾入队，队头出队。

出了基于顺序表的实现，队列还可以基于链表。

在顺序队列入队和出队的过程中，会发生这种情况：假设队列长度为5，连续入队5个元素后，tail移动带了队尾，这时候无法入队，连续出队两个元素，head向后移动两个位置，其前有两个空位，队列不满，但由于tail移动了队尾，此时依然无法入队。此时我们可以采用数据迁移，将后面的数据按次序移动到队头，但是考虑的到每次执行出队操作时都需要搬移后所有的元素，此时出队操作的时间复杂度从O（1）变为了O（n），为了降低时间复杂度，我们只在tail到达队尾时（即例子中给出的情况）再执行数据搬移操作。具体实现有enqueue_2实现。

使用优化后的入队操作，虽然可以解决了例子中的情况，但在该情况下的入队操作时间复杂度变成了O（n），使用循环队列可以有效决定例子中的问题，而且时间复杂度为O(1)。
